Access Pattern
Controls which keys are chosen at each step. Affects how realistically each tree's structure mirrors real-world workloads.
Each key is drawn uniformly from [1…N]. No hot spots. Baseline distribution — tests average-case behaviour.
Keys inserted as 1, 2, 3 … N in order. Worst case for naïve BSTs; shows how AVL/RB prevent O(n) degeneration.
80 % of operations target the hottest 20 % of keys (Pareto / 80-20 rule). Favours Splay, which self-organises toward frequently accessed nodes.
Operation Under Test
The single operation type that is timed and counted. Each option benchmarks a full batch of N operations of that type — not just one.
Inserts all N keys into an empty tree. Measures raw build time and shows how the tree balances as it grows.
Searches for each of the N keys in a pre-built tree (all N keys are inserted first, then N searches are timed). Tests read-path performance.
Deletes each of the N keys from a pre-built tree (all N inserted first, then N deletions are timed). Tests restructuring cost on removal.
Note — Lookup & Delete pre-build For Lookup and Delete, the tree is fully populated with all N keys before timing starts. Only the timed operation phase (N searches or N deletions) is measured — not the initial insert phase.

No balance field — accessed node is rotated to root (zig/zig-zig/zig-zag). Amortized O(log n). Naturally caches hot keys near root.
B-tree of order 4. Each node holds 1–3 keys and 2–4 children. Perfectly balanced by height. Splits on overflow, merges/borrows on underflow.
Randomized BST. Each node carries a random priority forming a max-heap. No explicit balancing — random priorities keep expected height O(log n).
Simplified Red-Black variant (Arne Andersson, 1993). Red links only on right spine; skew + split rebalancing. Cleaner invariants than RB with similar performance.
